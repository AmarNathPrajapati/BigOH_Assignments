1. Entities and Their Attributes


1️⃣ User
📌 Attributes:

UserID (PK) – Unique identifier
Username – Unique username
Email – User email (for login & notifications)
FullName – User's full name
Bio – Short description
ProfilePicture – URL of profile pic
PhoneNumber – Optional
DateOfBirth – Age verification
CreatedAt – Account creation date
UpdatedAt – Last profile update 

📌 Methods:

registerUser() – Naya user create kare
loginUser() – Authentication handle kare
updateProfile() – Profile update kare
followUser(UserID) – Kisi user ko follow kare
unfollowUser(UserID) – Kisi user ko unfollow kare------------------------------------------------->Done


2️⃣ Post
📌 Attributes:

PostID (PK) – Unique identifier
UserID (FK) – Post kis user ka hai
ImageURL – Image/video URL
Caption – Post ka description
Location – Optional, post ka location
CreatedAt – Post creation date
UpdatedAt – Last edited date
📌 Methods:

createPost(UserID, ImageURL, Caption)
deletePost(PostID)
editPost(PostID, newCaption)------------------------------------------------->Done


3️⃣ Comment
📌 Attributes:

CommentID (PK) – Unique identifier
PostID (FK) – Kis post pe comment hua
UserID (FK) – Kis user ne comment kiya
Text – Comment ka content
CreatedAt – Comment date
📌 Methods:

addComment(PostID, UserID, Text)
deleteComment(CommentID)------------------------------------------------->Done



4️⃣ Like
📌 Attributes:

LikeID (PK) – Unique identifier
PostID (FK) – Kis post pe like hua
UserID (FK) – Kis user ne like kiya
CreatedAt – Like date
📌 Methods:

likePost(PostID, UserID)
unlikePost(LikeID)------------------------------------------------->Done


5️⃣ Story
📌 Attributes:

StoryID (PK) – Unique identifier
UserID (FK) – Kis user ki story hai
MediaURL – Story content
ExpiryTime – 24 hours ke baad expire hone ka time
CreatedAt – Story upload date
📌 Methods:

addStory(UserID, MediaURL)
deleteStory(StoryID)------------------------------------------------->Done


6️⃣ Follow Relationship
📌 Attributes:

FollowID (PK) – Unique identifier
FollowerID (FK) – Kaun follow kar raha hai
FollowingID (FK) – Kise follow kiya ja raha hai
CreatedAt – Follow date
📌 Methods:

followUser(FollowerID, FollowingID)
unfollowUser(FollowerID, FollowingID)------------------------------------------------->Done

7️⃣ Message (DMs)
📌 Attributes:

MessageID (PK) – Unique identifier
SenderID (FK) – Sender ka UserID
ReceiverID (FK) – Receiver ka UserID
Text – Message ka content
CreatedAt – Message send time
📌 Methods:

sendMessage(SenderID, ReceiverID, Text)
deleteMessage(MessageID)------------------------------------------------->Done



2. Relationship Between Entities
Entity	Relationship	Entity
User ↔ Post	One-to-Many	(Ek user multiple posts kar sakta hai)
User ↔ Comment	One-to-Many	(Ek user multiple comments kar sakta hai)
User ↔ Like	One-to-Many	(Ek user multiple likes kar sakta hai)
User ↔ Story	One-to-Many	(Ek user multiple stories share kar sakta hai)
User ↔ Follow	Many-to-Many	(Users ek dusre ko follow kar sakte hain)
User ↔ Message	Many-to-Many	(Users ek dusre ko message kar sakte hain)
Post ↔ Comment	One-to-Many	(Ek post pe multiple comments ho sakte hain)
Post ↔ Like	One-to-Many	(Ek post pe multiple likes ho sakte hain)


3. Data Flow and Communication Between Entities
User Registration/Login:
User entity me authentication ke liye email/password store hoga
Post Creation:
UserID foreign key ke through Post entity se connected hoga
Like and Comment System:
User entity ke UserID ko Like aur Comment entity me foreign key rakha jayega
Follow System:
Follow table FollowerID aur FollowingID ko track karega
Message System (DMs):
UserID Message entity se connected hoga
4. Authentication Mechanism
User Registration – User email/password se register karega
Password Hashing – Secure password storage ke liye bcrypt ya Argon2 use hoga
JWT Authentication – User session ko maintain karne ke liye JWT token use kiya jayega
OAuth Integration – Google/Facebook login ka option diya ja sakta hai
Two-Factor Authentication (2FA) – Extra security ke liye phone/email verification
5. ER Diagram Representation
ER Diagram me entities ko boxes, attributes ko ovals, aur relationships ko diamond shapes se represent karenge.

👨‍💻 Main Elements in ER Diagram:

User – Central entity
Post, Comment, Like – Content-related entities
Follow – User relationship track karega
Message – DMs ko handle karega
Agar aap ER diagram banana chahte ho to MySQL Workbench, Lucidchart, DB Diagram, ya Draw.io use kar sakte ho.
